# 开发日志 — 成员4（拓展功能 + Docker部署）

## 项目概述
- 项目名称: 直播助手网站 (Live Assistant)
- 团队规模: 4人
- 技术栈: Vue3 + Spring Boot + SQLite + 阿里云TTS
- 开发周期: 4天 (2026-07-04 ~ 2026-07-07)

我（成员4）负责的模块:
- 声音评价 (Feedback)
- 直播统计 (Stats)
- 脚本管理 (Scripts)
- 帮助中心 (Help)
- JWT认证配置
- Docker Compose + Nginx 部署配置

---

## Day 1 (7月4日): 技术选型与项目初始化

### 完成的工作
1. 参与4人团队技术选型讨论，最终决议:
   - 后端分3个微服务: backend-main(主服务8081)、backend-voice-ai(语音AI8082)、backend-extended(拓展功能8083)
   - 数据库: SQLite (轻量，适合小学期快速开发)
   - ORM: MyBatis + 注解SQL (不用XML)
   - 认证: JWT (无状态，跨服务友好)
   - 部署: Docker Compose + Nginx反向代理

2. 创建 backend-extended 模块骨架:
   - pom.xml (spring-boot-starter-web, mybatis-spring-boot-starter, sqlite-jdbc, jjwt, lombok)
   - BackendExtendedApplication.java
   - application.yml (端口8083, SQLite数据源)

3. 在 backend-main 的 schema.sql 中新增4张表:
   - scripts (脚本表)
   - feedbacks (评价表)
   - live_sessions (直播会话表)
   - help_articles (帮助文章表)

### 遇到的问题
- SQLite的datetime默认值与MySQL不同，需要确认CURRENT_TIMESTAMP的兼容性
- 解决方案: SQLite 3.35+ 支持 CURRENT_TIMESTAMP，确认docker镜像内版本满足要求

### 技术决策
- 拓展功能独立为微服务 backend-extended，与主服务和AI服务解耦
- 所有SQL用MyBatis注解方式，不引入XML映射文件，减少配置复杂度
- JWT密钥统一配置在 application.yml 中

## Day 2 (7月5日): Entity + Mapper + Service 开发

### 完成的工作
1. 创建4个Entity:
   - Script.java: id, userId, title, content, category, createdAt, updatedAt
   - Feedback.java: id, userId, voiceModelId, rating(1-5 CHECK约束), comment, createdAt
   - LiveSession.java: id, userId, scriptId, platform, startTime, endTime, statsData(TEXT存JSON)
   - HelpArticle.java: id, title, content, category, sortOrder, createdAt

2. 创建4个Mapper (全注解SQL):
   - ScriptMapper.java: CRUD + 按分类/关键词搜索
   - FeedbackMapper.java: CRUD + 按声音模型ID查询 + 平均评分统计
   - LiveSessionMapper.java: CRUD + 按用户ID+时间范围查询
   - HelpArticleMapper.java: 查询全部/按分类/按ID/模糊搜索 + INSERT

3. 创建4个Service:
   - ScriptService.java: 脚本CRUD业务逻辑
   - FeedbackService.java: 评价提交+声音评分统计
   - StatsService.java: 直播时长/场次统计+综合仪表盘数据
   - HelpService.java: 帮助文章分类查询+关键词搜索

### 遇到的问题
- MyBatis下划线自动转驼峰问题：entity字段 sortOrder vs 数据库列 sort_order
- 解决方案: application.yml 配置 `map-underscore-to-camel-case: true`

### 技术决策
- LiveSession.statsData 用 TEXT 类型存储JSON字符串，避免为统计数据单独建表
- Feedback的rating字段用 CHECK 约束确保值在1-5之间
- 所有Mapper使用 @Mapper 注解自动扫描，不配置 @MapperScan

## Day 3 (7月6日): Controller + JWT认证 + 联调

### 完成的工作
1. JWT认证配置:
   - JwtUtil.java: 生成token、解析token、验证token（HMAC-SHA256）
   - AuthInterceptor.java: 实现 HandlerInterceptor，从 Authorization header 取token验证
   - WebConfig.java: 注册拦截器，放行 /api/help/ 路径（帮助文章无需登录）

2. 创建4个Controller (14个API端点):
   - ScriptsController.java (4个):
     - POST /api/scripts — 创建脚本
     - GET /api/scripts — 获取用户脚本列表
     - GET /api/scripts/{id} — 获取脚本详情
     - PUT /api/scripts/{id} — 更新脚本
     - DELETE /api/scripts/{id} — 删除脚本
   - FeedbackController.java (4个):
     - POST /api/feedback — 提交评价
     - GET /api/feedback/voice/{voiceModelId} — 获取声音评价列表
     - GET /api/feedback/voice/{voiceModelId}/rating — 获取平均评分
     - DELETE /api/feedback/{id} — 删除评价
   - StatsController.java (2个):
     - GET /api/stats/dashboard — 仪表盘综合统计
     - GET /api/stats/sessions — 直播会话统计
   - HelpController.java (3个):
     - GET /api/help/articles — 帮助文章列表(支持?category=参数)
     - GET /api/help/articles/{id} — 文章详情
     - GET /api/help/search?keyword= — 关键词搜索

3. 编写 Docker Compose 和 Nginx 配置:
   - docker-compose.yml: 4个服务(nginx/backend-main/backend-voice-ai/backend-extended)，统一网络taffy-network
   - nginx.conf: 前端静态文件 + API反向代理，/api/feedback//api/stats//api/scripts//api/help/ 全部代理到 backend-extended:8083

### 遇到的问题
- AuthInterceptor中如何从请求中获取用户ID传给Controller
- 解决方案: 使用 request.setAttribute("userId", userId)，Controller通过 @RequestAttribute 获取

### 技术决策
- /api/help/ 路径不需要登录认证，在WebConfig中单独放行
- 统计模块复用 feedbacks 和 live_sessions 表的数据做聚合查询
